# Icon Assets

Place your master icon source file here and generate the required platform outputs.

## Required files

```
assets/icons/
├── icon.png          # Master source – minimum 1024×1024, square, transparent background
├── icon.ico          # Windows (multi-size ICO)
├── icon.icns         # macOS (Apple icon container)
└── tray/
    ├── trayTemplate.png   # macOS tray – 16×16 white/transparent (+ @2x at 32×32)
    ├── trayTemplate@2x.png
    ├── tray-linux.png     # Linux/Windows tray – 22×22 or 32×32 RGBA
    └── tray-linux@2x.png  # 44×44 or 64×64
```

## Generating outputs from a master PNG

Install ImageMagick and `icnsutils` (or `png2icns`) once:

```bash
# macOS
brew install imagemagick

# Linux / CI
sudo apt-get install -y imagemagick icnsutils
```

### Windows – icon.ico (multi-size)

```bash
convert assets/icons/icon.png \
  \( -clone 0 -resize 16x16   \) \
  \( -clone 0 -resize 24x24   \) \
  \( -clone 0 -resize 32x32   \) \
  \( -clone 0 -resize 48x48   \) \
  \( -clone 0 -resize 64x64   \) \
  \( -clone 0 -resize 128x128 \) \
  \( -clone 0 -resize 256x256 \) \
  -delete 0 assets/icons/icon.ico
```

### macOS – icon.icns

```bash
mkdir -p /tmp/fluxer.iconset
for s in 16 32 64 128 256 512; do
  convert assets/icons/icon.png -resize ${s}x${s}     /tmp/fluxer.iconset/icon_${s}x${s}.png
  convert assets/icons/icon.png -resize $((s*2))x$((s*2)) /tmp/fluxer.iconset/icon_${s}x${s}@2x.png
done
iconutil -c icns /tmp/fluxer.iconset -o assets/icons/icon.icns   # macOS only
# Linux alternative: png2icns assets/icons/icon.icns /tmp/fluxer.iconset/*.png
```

### Tray icons

macOS tray icons must be named `*Template*` so the OS applies automatic light/dark tinting:

```bash
convert assets/icons/icon.png -resize 16x16 assets/icons/tray/trayTemplate.png
convert assets/icons/icon.png -resize 32x32 assets/icons/tray/trayTemplate@2x.png
```

Linux/Windows tray icons use the coloured version:

```bash
convert assets/icons/icon.png -resize 22x22 assets/icons/tray/tray-linux.png
convert assets/icons/icon.png -resize 44x44 assets/icons/tray/tray-linux@2x.png
```

### Flatpak icons (generated by CI automatically)

The CI workflow generates these; you do not need them locally unless testing Flatpak builds:

```bash
for s in 16 32 48 64 128 256 512; do
  convert assets/icons/icon.png -resize ${s}x${s} flatpak/icons/icon_${s}.png
done
```

## Build commands

### Install dependencies

```bash
npm ci
```

### Development (live reload)

```bash
npm run dev
```

### Production builds

| Platform | Command | Output |
|----------|---------|--------|
| Windows  | `npm run build:win`   | `release/*.exe` (NSIS installer + portable) |
| macOS    | `npm run build:mac`   | `release/*.dmg` (universal arm64 + x64) |
| Linux    | `npm run build:linux` | `release/*.tar.gz` |

> **Code signing** – set `CSC_LINK` / `CSC_KEY_PASSWORD` (Windows/macOS) environment variables
> before building, or the build will produce unsigned binaries.

> **macOS notarisation** – also set `APPLE_ID`, `APPLE_APP_SPECIFIC_PASSWORD`, `APPLE_TEAM_ID`.

### Build all platforms at once

```bash
npm run build
```

## Protocol registration (Linux tar.gz, non-Flatpak)

After extracting the tar.gz, register the `fluxerworld://` URL scheme:

```bash
# Copy desktop file
mkdir -p ~/.local/share/applications
cp /path/to/fluxer-world/resources/fluxer-world.desktop ~/.local/share/applications/

# Register MIME handler
xdg-mime default org.fluxer.World.desktop x-scheme-handler/fluxerworld

# Refresh desktop database
update-desktop-database ~/.local/share/applications
```

The Flatpak package handles protocol registration automatically via the `.desktop` file.
